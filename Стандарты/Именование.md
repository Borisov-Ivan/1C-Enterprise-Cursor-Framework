# Стандарт именования

**Версия:** 1.0  
**Приоритет:** Критичный  
**Источник:** Глава 5. Объекты конфигурации, стандарты 1С:ИТС

---

## Правила именования

### Объекты метаданных

**Правило:** Использовать префикс проекта для всех объектов метаданных.

```bsl
// ❌ НЕПРАВИЛЬНО: Без префикса
Справочник.Товары
Документ.Заказ

// ✅ ПРАВИЛЬНО: С префиксом проекта
Справочник.ргТовары
Документ.ргЗаказ
```

**Источник:** Стандарт #std453 (1С:ИТС)

---

### Переменные

**Правило:** CamelCase, русский язык, без транслитерации.

```bsl
// ❌ НЕПРАВИЛЬНО: Транслитерация
Product = Новый СправочникСсылка.Товары;

// ✅ ПРАВИЛЬНО: Русский язык, CamelCase
ТоварСсылка = Новый СправочникСсылка.Товары;
```

---

### Функции и процедуры

**Правило:** ПаскальКейс, начинается с глагола.

```bsl
// ❌ НЕПРАВИЛЬНО: Существительное
Функция ДанныеТовара() Экспорт

// ✅ ПРАВИЛЬНО: Глагол
Функция ПолучитьДанныеТовара() Экспорт
```

```bsl

ПараметрыСеанса.ТекущийПользователь
```

### Роли

**Правило:** Две схемы именования:
1. Прикладные роли - от названия должности (Бухгалтер, Кассир, Администратор)
2. Роли для тонкой настройки - от описания разрешаемого действия

```bsl
// ✅ ПРАВИЛЬНО: Прикладные роли
Роль "Бухгалтер"
Роль "Кассир"
Роль "Администратор"

// ✅ ПРАВИЛЬНО: Роли для тонкой настройки
Роль "ДобавлениеИзменениеНСИ"
Роль "ЧтениеДополнительныхСведений"
Роль "ИнтерактивноеОткрытиеВнешнихОтчетовИОбработок"
```

**Источник:** Раздел 2. Имена объектов метаданных (строки 76-77)

---

### Справочники

**Правило:** Во множественном числе, от описания списка объектов.

```bsl
// ❌ НЕПРАВИЛЬНО: Единственное число, префикс "Справочник"
Справочник "Валюта"
Справочник "Товар"

// ✅ ПРАВИЛЬНО: Множественное число, без префикса
Справочник "Валюты"
Справочник "Товары"
Справочник "ГруппыИсполнителейЗадач"
Справочник "Пользователи"
```

**Источник:** Раздел 2. Имена объектов метаданных (строка 142)

```bsl
Документ "НакладнаяТоваров"
Документ "АктСверки"

// ✅ ПРАВИЛЬНО: Без префикса, лаконичное имя
Документ "ЗаказПокупателя"

Документ "СверкаВзаиморасчетов"
```

### Функциональные опции

**Правило:** От описания включаемой функциональности, для типа Булево - от неопределенной формы глагола.

```bsl
// ✅ ПРАВИЛЬНО
ФункциональнаяОпция "ИспользоватьБизнесПроцессыИЗадачи"
ФункциональнаяОпция "ИспользоватьВерсионированиеОбъектов"
```

**Источник:** Раздел 2. Имена объектов метаданных (строки 81-87)

---

### Константы

**Правило:** 
- Если связана с функциональной опцией - созвучно функциональной опции
- Если тип Булево - от неопределенной формы глагола
- В остальных случаях - от описания объекта, значение которого хранит

```bsl
// ✅ ПРАВИЛЬНО
Константа "ТипХраненияФайлов"
Константа "НастройкаПроксиСервера"
Константа "ВыполнятьПроверкуЭЦПНаСервере"
Константа "ИзвлекатьТекстыФайловНаСервере"
```

**Источник:** Раздел 2. Имена объектов метаданных (строки 124-140)

### Определяемые типы

**Правило:** В единственном числе, от назначения. Не использовать слова "Тип", "Объект", "Ссылка", не называть как типы данных.

```bsl
// ❌ НЕПРАВИЛЬНО: Слова "Тип", "Объект", названия типов данных
ОпределяемыйТип "ТипСтрока"
ОпределяемыйТип "СсылкаНаОбъект"
ОпределяемыйТип "Строка"

// ✅ ПРАВИЛЬНО: От назначения
ОпределяемыйТип "АртикулНоменклатуры"
ОпределяемыйТип "КонтактВзаимодействий"
```

**Источник:** Раздел 2. Имена объектов метаданных (строки 88, 100-102)

## Чек-лист проверки

- [ ] Все объекты метаданных имеют префикс проекта (если требуется)
- [ ] Переменные используют CamelCase
- [ ] Функции начинаются с глагола
- [ ] Нет транслитерации в именах
- [ ] Справочники во множественном числе
- [ ] Документы в единственном числе
- [ ] Регистры сведений/накопления во множественном числе
- [ ] Отчеты и обработки без префиксов "Отчет", "Обработка"
- [ ] Функциональные опции от неопределенной формы глагола (для Булево)
- [ ] Регламентные задания от существительного
- [ ] Подписки на события от неопределенной формы глагола
- [ ] Общие формы без префиксов "Форма", "Диалог"
- [ ] Web-сервисы на английском языке
