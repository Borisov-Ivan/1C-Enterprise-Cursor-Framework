# Стандарт комментирования кода

**Версия:** 1.0  
**Приоритет:** Средний  
**Источник:** Стандарты 1С:ИТС, корпоративные стандарты

---

## Правило: Комментировать экспортные функции

**Требование:** Все экспортные функции ДОЛЖНЫ иметь комментарии с описанием, параметрами и возвращаемым значением.

```bsl
// ❌ НЕПРАВИЛЬНО: Нет комментария
Функция ПодписатьДанные(Данные, ОтпечатокСертификата) Экспорт
    // Код функции
КонецФункции

// ✅ ПРАВИЛЬНО: Полный комментарий
// Подписывает данные электронной подписью с использованием указанного сертификата.
//
// Параметры:
//   Данные - ДвоичныеДанные - данные для подписи
//   ОтпечатокСертификата - Строка - SHA-1 отпечаток сертификата
//
// Возвращаемое значение:
//   Структура - результат операции:
//     * Успех - Булево - признак успешного выполнения
//     * Подпись - ДвоичныеДанные - подписанные данные
//     * Ошибка - Строка - описание ошибки (если Успех = Ложь)
//
Функция ПодписатьДанные(Данные, ОтпечатокСертификата) Экспорт
    // Код функции
КонецФункции
```

## Правило: Обрамлять вставки в типовой код

**Требование:** Все вставки кода в типовые модули ДОЛЖНЫ быть обрамлены маркерами.

```bsl
// ❌ НЕПРАВИЛЬНО: Нет маркеров
Если ПолучитьФункциональнуюОпцию("ргИспользоватьСервисCryptoAPI") Тогда
    АктивныеЭкземпляры = ргОбщегоНазначенияВызовСервера.ПолучитьЭкземплярыCryptoAPI();
КонецЕсли;

// ✅ ПРАВИЛЬНО: Маркеры начала и конца
//РГИТС БорисовИ.Г. Подсистема ргСервисКриптографииCryptoAPI 2025-01-XX
Если ПолучитьФункциональнуюОпцию("ргИспользоватьСервисCryptoAPI") Тогда
    АктивныеЭкземпляры = ргОбщегоНазначенияВызовСервера.ПолучитьЭкземплярыCryptoAPI();
КонецЕсли;
//Конец вставки РГИТС
```

## Чек-лист проверки

- [ ] Все экспортные функции имеют комментарии
- [ ] Комментарии содержат описание, параметры и возвращаемое значение
- [ ] Вставки в типовой код обрамлены маркерами
- [ ] Комментарии написаны на русском языке
