# Стандарт разработки форм и интерфейса

**Версия:** 1.0  
**Приоритет:** Высокий  
**Источник:** Раздел 13. Формы и интерфейс (строки 3800-6200)

---

## Описание

Стандарт определяет правила разработки форм и интерфейса в конфигурации 1С:Предприятие. Включает требования к структуре модулей форм, обработчикам событий, размерам экрана и оформлению элементов интерфейса.

---

## Структура модулей форм

**Правило:** Модуль формы должен быть структурирован по разделам: обработчики событий формы, обработчики событий элементов, обработчики команд, служебные процедуры и функции.

```bsl
// ✅ ПРАВИЛЬНО: Структурированный модуль формы
#Область ОбработчикиСобытийФормы

&НаКлиенте
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    // Инициализация формы
КонецПроцедуры

&НаКлиенте
Процедура ПриОткрытии(Отказ)
    // Обработка открытия формы
КонецПроцедуры

#КонецОбласти

#Область ОбработчикиСобытийЭлементовШапкиФормы

&НаКлиенте
Процедура ПолеНаименованиеПриИзменении(Элемент)
    // Обработка изменения поля
КонецПроцедуры

#КонецОбласти

#Область ОбработчикиСобытийЭлементовТаблицыФормыТовары

&НаКлиенте
Процедура ТоварыПриИзменении(Элемент)
    // Обработка изменения табличной части
КонецПроцедуры

#КонецОбласти

#Область ОбработчикиКомандФормы

&НаКлиенте
Процедура Заполнить(Команда)
    ЗаполнитьНаСервере();
КонецПроцедуры

#КонецОбласти

#Область СлужебныеПроцедурыИФункции

&НаСервере
Процедура ЗаполнитьНаСервере()
    // Реализация заполнения
КонецПроцедуры

#КонецОбласти
```

**Источник:** Раздел 13. Структура модулей форм (строки 3995-4011)

## Обработчики событий элементов формы

**Правило:** У каждого события должна быть назначена своя процедура-обработчик. Если одинаковые действия выполняются при событиях разных элементов, следует создать отдельную процедуру и вызывать ее из обработчиков.

```bsl
// ❌ НЕПРАВИЛЬНО: Вызов одного обработчика из другого
&НаКлиенте
Процедура ПоИсполнителюПриИзменении(Элемент)
    ПараметрыОтбора = Новый Соответствие();
    ПараметрыОтбора.Вставить("ПоАвтору", ПоАвтору);
    ПараметрыОтбора.Вставить("ПоИсполнителю", ПоИсполнителю);
    УстановитьОтборСписка(Список, ПараметрыОтбора);
КонецПроцедуры

&НаКлиенте
Процедура ПоАвторуПриИзменении(Элемент)
    ПоИсполнителюПриИзменении(Неопределено); // Неправильно: вызов обработчика
КонецПроцедуры

// ✅ ПРАВИЛЬНО: Отдельная процедура для общей логики
&НаКлиенте
Процедура ПоИсполнителюПриИзменении(Элемент)
    УстановитьОтбор();
КонецПроцедуры

&НаКлиенте
Процедура ПоАвторуПриИзменении(Элемент)
    УстановитьОтбор();
КонецПроцедуры

&НаСервере
Процедура УстановитьОтбор()
    ПараметрыОтбора = Новый Соответствие();
    ПараметрыОтбора.Вставить("ПоАвтору", ПоАвтору);
    ПараметрыОтбора.Вставить("ПоИсполнителю", ПоИсполнителю);
    УстановитьОтборСписка(Список, ПараметрыОтбора);
КонецПроцедуры
```

**Источник:** Раздел 13. Обработчики событий элементов формы (строки 4051-4066)

## Размеры экрана

**Правило:** При проектировании интерфейсов типовым разрешением экрана следует считать 1024х768 пикселей. Разработку нужно вести в стандартном разрешении - 96 DPI.

**Условия эксплуатации:**
- Основное окно программы растянуто на весь экран
- Панель задач операционной системы видна, расположена в нижней части экрана и уменьшает высоту на 40 пикселей
- Программа открыта в браузере. Высота рабочей области сокращается из-за заголовков и панелей браузера в среднем на 60 пикселей

**Доступная область:** 1024х668 пикселей

**Требования:**
- Элементы внутри форм должны помещаться без вертикальной и горизонтальной полос прокрутки
- Исключение: списки, в которых вертикальная полоса прокрутки допускается, а горизонтальная - нежелательна

**Источник:** Раздел 13. Размеры экрана (строки 6174-6186)

## Оформление групп разделов

**Правило:** Группа раздела может включать в себя элементы управления, подчиненные группы разделов и другие группы разделов.

**Требования к оформлению:**
- Группы разделов должны быть логически структурированы
- Рекомендуется использовать вложенные группы для сложных форм
- Элементы управления должны быть сгруппированы по функциональному назначению

**Источник:** Раздел 13. Оформление групп разделов (строки 6188-6192)

## Обработчики команд формы

**Правило:** Обработчики команд формы размещаются в отдельном разделе модуля формы.

```bsl
#Область ОбработчикиКомандФормы

&НаКлиенте
Процедура Заполнить(Команда)
    ЗаполнитьНаСервере();
КонецПроцедуры

&НаКлиенте
Процедура Провести(Команда)
    ПровестиНаСервере();
КонецПроцедуры

#КонецОбласти
```

**Источник:** Раздел 13. Обработчики команд формы (строки 3995-4011)

## Чек-лист проверки

- [ ] Модуль формы структурирован по разделам (обработчики событий формы, элементов, команд, служебные процедуры)
- [ ] У каждого события назначена своя процедура-обработчик
- [ ] Общая логика вынесена в отдельные процедуры
- [ ] Формы разработаны для разрешения 1024х768 пикселей
- [ ] Элементы помещаются без горизонтальной прокрутки
- [ ] Группы разделов логически структурированы
- [ ] Обработчики команд размещены в отдельном разделе
