# Стандарт разделения клиентского и серверного кода

**Версия:** 1.0  
**Приоритет:** Высокий  
**Источник:** Глава 4. Встроенный язык

---

## Правило: Использовать обёртки для серверных модулей

**Проблема:** Прямые вызовы серверных модулей из клиентского кода нарушают архитектуру.

```bsl
// ❌ НЕПРАВИЛЬНО: Прямой вызов серверного модуля
&НаКлиенте
Процедура ОбработкаНажатия()
    Результат = ргСервисКриптографииCryptoAPI.ПроверитьДоступностьСервиса(); // ОШИБКА: прямой вызов сервера
КонецПроцедуры

// ✅ ПРАВИЛЬНО: Использование обёртки вызова сервера
&НаКлиенте
Процедура ОбработкаНажатия()
    Результат = ргСервисКриптографииCryptoAPIВызовСервера.ПроверитьДоступностьСервиса();
КонецПроцедуры
```

## Правило: Использовать ПараметрыПриложения для клиентского кэша

**Проблема:** Модули Server+ClientOrdinaryApplication запрещают переменные модуля.

```bsl
// ❌ НЕПРАВИЛЬНО: Переменная модуля в Server+ClientOrdinaryApplication
Перем КэшОпроса; // ОШИБКА: запрещено

// ✅ ПРАВИЛЬНО: Использование ПараметрыПриложения
Функция ПолучитьКэшОпроса()
    Кэш = ПараметрыПриложения.Получить("КэшДанныхОпроса");
    Если Кэш = Неопределено Тогда
        Кэш = Новый Соответствие;
        ПараметрыПриложения.Установить("КэшДанныхОпроса", Кэш);
    КонецЕсли;
    Возврат Кэш;
КонецФункции
```

## Чек-лист проверки

- [ ] Нет прямых вызовов серверных модулей из клиентского кода
- [ ] Используются обёртки вызова сервера (модули *ВызовСервера)
- [ ] Клиентский кэш использует ПараметрыПриложения
- [ ] Нет переменных модуля в Server+ClientOrdinaryApplication
